# Inventário Familiar

A self-hosted portal for families going through an estate/inheritance process
(*inventário*). One place for the documents everyone needs to see and for the
money nobody wants to argue about.

Built for a real use case: a property sold in installments, monthly payments
coming in, a lawyer entitled to a percentage, recurring estate expenses, and
heirs who all deserve a clear, individual view of what is theirs.

> The user interface is in Portuguese: the product is built for its market.

## What it does

- **Shared documents**: every member sees all estate documents (any file
  format; S3 storage with a database fallback for small deployments).
- **Automatic income splitting**: when an income entry (*receita*) is
  registered, the engine takes the lawyer's fee off the top, deducts the
  expenses due that month, and splits the net equally among active heirs.
  Leftover cents go one by one to the first heirs; the resulting shares are
  stored as an immutable history. `computeSplit` is a pure function with a
  full test suite.
- **Individual statements**: each member sees only their own credits and
  payouts. The admin sees everything and can "view as" any member, with the
  impersonation recorded in the audit log.
- **Payouts and expenses**: payments to members are registered against each
  income entry; expenses are deducted from the month they are due.
- **Payment keys**: each member registers their own PIX key; pending keys are
  flagged until they do.
- **Audit log**: every login, download, entry and impersonation is recorded
  with IP and user agent.
- **Install as app**: PWA manifest plus an in-app guide for iOS and Android.

## Security model

- First access with a one-time code (delivered via WhatsApp Cloud API if
  configured, or generated by the admin and sent personally); the member then
  creates a password (bcrypt).
- After that, phone + password. Sessions are JWTs in an httpOnly cookie.
- Password reset is admin-only and **blocks the account immediately** while
  revoking every active session (a session epoch bump), so a lost phone is
  one admin action away from being locked out.
- Unregistered phone numbers are told, explicitly, that they have no access.
- The admin is a regular member with extra powers: set
  `INVENTARIO_ADMIN_PHONE` to the phone of the member who runs the estate.

## Getting started

```bash
npm install
cp .env.example .env    # fill DATABASE_URL, AUTH_SECRET, INVENTARIO_ADMIN_PHONE
npm run db:init         # creates the tables
npm run db:seed-demo    # optional: fictional demo members
npm run dev
```

Log in with a seeded member: ask the "admin" flow to generate a first-access
code (Members tab) or insert one directly while exploring locally.

## Architecture

- **Next.js 16 / React 19**, single deployable app (UI + API routes).
- **PostgreSQL** with a plain SQL schema ([scripts/init.sql](scripts/init.sql));
  Prisma models map onto those tables for typed access.
- **The financial engine** lives in
  [lib/inventario/repo.ts](lib/inventario/repo.ts): splitting, monthly
  expense composition, recalculation guards (an entry that already has
  payouts cannot be silently recomputed).
- **Storage**: S3 (SigV4 signing, no SDK dependency) with a base64 database
  fallback under 8MB.

## Testing

```bash
npm run typecheck
npm test   # vitest: the split engine (fees, leftover cents, expenses)
```
